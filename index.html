MCPs — was ist was?

Drei Wege, Claude Code zu erweitern. Sie heißen ähnlich, sie tun ähnliches — und doch sind sie klar verschieden. Diese Seite erklärt in einem Sitz, welcher Weg wann der richtige ist.

MCP
⚙️

MCP-Server

Tools zur Laufzeit

Ein externer Prozess, der Claude über das offene Model Context Protocol Werkzeuge anbietet — eine API anrufen, eine Datenbank lesen, ein Browser steuern. Claude entscheidet selbst, wann es das Werkzeug benutzt.

# MCP installieren (Beispiel: Chrome DevTools)
claude mcp add chrome-devtools \
  -- npx -y chrome-devtools-mcp@latest
stdio / HTTP JSON-RPC live tools

Skill
📖

Agent Skills

Playbook das Claude liest

Ein Markdown-Ordner mit SKILL.md + Frontmatter. Claude erkennt am Description-Feld automatisch, wann der Skill relevant ist, und lädt nur dann den vollen Inhalt. Ideal für wiederkehrende Workflows.

# Eigenen Skill anlegen
mkdir -p ~/.claude/skills/git-summary
cat > ~/.claude/skills/git-summary/SKILL.md <<'EOF'
---
description: Summarize uncommitted git changes
---
Run `git diff HEAD` and summarize in 2-3 bullets.
EOF
SKILL.md Auto oder /name Markdown only

Plugins

Bündel zum Verteilen

Ein versioniertes Paket, das mehrere Skills + MCPs + Slash-Commands + Hooks zusammenfasst. Wird über einen Marketplace verteilt — ein GitHub-Repo, eine URL, oder Anthropics offiziellen Plugin-Store.

# Plugin aus offiziellem Marketplace installieren
/plugin install frontend-design@claude-plugins-official

Direktvergleich

Sechs Achsen auf denen MCPs, Skills und Plugins sich unterscheiden.

⚙️ MCP 📖 Skill 📦 Plugin
Was ist es Externer Tool-Server (JSON-RPC über stdio/HTTP) Markdown-Playbook mit SKILL.md Versioniertes Bündel: Skills + MCPs + Commands + Hooks
Wann benutzen Claude braucht Daten oder Aktionen in Echtzeit Wiederkehrender Workflow, Checkliste, Domänen-Wissen Mehreres davon zusammen verteilen — Team, Community
Wer liefert Drittanbieter, Anthropic, eigene Server Du, dein Team, projekt-lokal Du, Anthropic, Community via Marketplaces
Installation claude mcp add <name> -- <cmd> ~/.claude/skills/<name>/SKILL.md /plugin install <name>@<marketplace>
Aufruf Automatisch wenn relevant Auto (Description-Match) oder /<name> /<plugin>:<skill> oder Tools werden registriert
Lebt wo Eigener Prozess + Eintrag in ~/.claude.json Datei: SKILL.md in Skill-Verzeichnis ~/.claude/plugins/ + Marketplace-Eintrag
